# Prompt 00 — Project Brief

あなたは熟練のゲームボーイ開発者兼C言語エンジニアです。Mac上でGBDK-2020を使い、最終的に実機Game Boy / Game Boy Color互換環境で動作するRogueクローンを作ってください。

## 目標

Palm Pilot版のiRogueで遊んでいた感触に近い、クラシックRogue系ゲームを作ります。参照資料としてはUNIX Rogue 5.4系を主に見ますが、完全なソース移植ではなく、Game Boyの制約に合わせた再実装です。

## 絶対に守る方針

- 「ローグライク風」ではなく、できるだけ元祖Rogueの構造を維持する。
- 最初はASCII/記号表示で実装する。
- 後から8x8ドット絵に差し替えられるよう、表示IDとゲーム内部IDを分離する。
- アイテムやモンスターの種類は原則として減らさない。
- 画面に全部屋を表示する必要はない。20x18タイル画面でスクロール表示する。
- 実機動作を最終目標にする。エミュレータだけで満足しない。
- GBDK-2020を使う。
- C言語で、メモリ使用量を常に意識する。

## 画面方針

Game Boy画面は160x144、8x8タイルで20x18マス。基本画面は次の構成にする。

- 下1行：常時ステータス欄
- その上：メッセージ表示。通常は直近1行だけ残す
- 必要時：2〜4行のメッセージポップアップを出す
- 残り：スクロールするダンジョン表示

## 操作方針

操作感は風来のシレンGBに近づける。Game Boyのボタン数に合わせ、以下を基本にする。

- 十字キー：4方向移動
- A：決定、隣接敵への攻撃、メニュー内決定
- B：キャンセル、メニューを閉じる
- Start：メインメニュー
- Select：持ち物
- A + 十字キー：方向指定、斜め移動や遠隔対象指定に使う候補
- B + 十字キー：高速移動または方向転換に使う候補

斜め移動はRogueの重要要素なので、削除しない。Game Boy向けの自然な入力方式を設計する。

## 重要システム

以下は必須です。

- ランダムダンジョン生成
- 部屋と通路
- 階段
- ターン制
- 視界、探索済み、未探索
- モンスターAI
- 戦闘
- 経験値とレベル
- HP、力、ACなどの基本ステータス
- 空腹
- 未識別アイテム
- 薬、巻物、杖、指輪、武器、防具、食料、金
- 呪い
- 罠
- 永久死
- 中断セーブ
- 死亡時の中断セーブ破棄
- ゲームログ

## 最初にやること

まずリポジトリ構成、ビルド環境、GBDK-2020の導入方法、最小ROMのビルド方法を設計してください。その後、実装計画を段階化してください。

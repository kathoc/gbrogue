# Prompt 04 — Acceptance Tests

各マイルストーンで、次の受け入れ条件を満たしているか確認してください。

## 共通

- `make clean && make` が通る
- ROMが生成される
- エミュレータで起動する
- コンパイル警告を放置しない
- メモリ使用量を記録する
- 実装した仕様と未実装仕様を `docs/status.md` に記録する

## 操作

- 十字キーで移動できる
- A/B/Start/Selectが設計どおりに動く
- メニューで戻れなくなる状態がない
- 斜め移動の操作が説明されている

## 表示

- 下1行のステータスが常時読める
- メッセージ1行が残る
- 重要なイベント時に2〜4行ポップアップを出せる
- ダンジョン表示がスクロールしても破綻しない
- ASCIIタイルの意味が分かる

## Rogueらしさ

- 未識別アイテムがある
- アイテムの正体を使うまで分からない
- 呪いがある
- 空腹がある
- 死んだら復活できない
- 罠がある
- モンスターがターン単位で動く
- 階層を降りる意味がある

## セーブ

- 中断セーブができる
- 再開できる
- 死亡後にセーブが消える
- 中断セーブが通常セーブのように悪用できない

## 実機向け

- フレーム落ちが気にならない
- LCDでも見える
- コントラストが低くても判別できる
- 起動からゲーム開始までが短い

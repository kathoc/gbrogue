# Prompt 02 — Architecture Design

このプロジェクトのアーキテクチャを設計してください。最終的には実機Game Boyで動かします。GBDK-2020、C言語、8x8タイル、ASCII表示から開始します。

## 重要方針

内部データと表示を分離してください。ゲーム内部ではモンスター、アイテム、地形をIDで管理し、表示はASCIIタイルIDにマッピングします。後から同じIDに8x8ドット絵を割り当てられるようにします。

## モジュール案

以下のような分割を基本にしてください。

- `main.c`：起動、メインループ
- `game.c/h`：ゲーム状態、ターン進行
- `map.c/h`：マップ、部屋、通路、階段
- `fov.c/h`：視界、探索済み管理
- `render.c/h`：画面描画、スクロール、ステータス、メッセージ
- `input.c/h`：Game Boy入力、メニュー操作
- `actor.c/h`：プレイヤー、モンスター
- `combat.c/h`：戦闘
- `items.c/h`：アイテム定義、識別、効果
- `inventory.c/h`：持ち物
- `rng.c/h`：乱数
- `save.c/h`：中断セーブ
- `ui_menu.c/h`：メニュー
- `assets/tiles_ascii.c`：ASCII風タイル
- `assets/tiles_graphic.c`：後の差し替え用

## データ構造の制約

Game BoyのWRAMは非常に小さいため、すべて固定長配列を基本にしてください。mallocは禁止。再帰も原則禁止。スタックを使いすぎないでください。

## マップ

Rogue風に80x24相当の論理マップを持つ。画面には一部だけをスクロール表示する。必要ならGame Boy制約に合わせて内部サイズを調整してよいが、Rogueの部屋・通路構造は維持する。

## セーブ

中断セーブを実装する。死亡時はセーブを消す。セーブが難しい段階ではスタブを作り、設計だけ先に用意する。

## 描画

最初はASCIIと同じ意味の記号タイルでよい。例：

- `@` プレイヤー
- `#` 通路
- `.` 床
- `+` ドア
- `|` `-` 壁
- `%` 階段または食料など、Rogue準拠で整理
- `!` 薬
- `?` 巻物
- `)` 武器
- `]` 防具
- `/` 杖
- `=` 指輪
- `*` 金または宝石
- 英字 モンスター

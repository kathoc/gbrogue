# Prompt 06 — Controls Specification

風来のシレンGBに近い操作感を目標に、Game Boy向け操作仕様を設計・実装してください。

## 基本

- 十字キー：移動
- A：決定、攻撃、目の前のものを調べる
- B：キャンセル、メッセージ送り
- Start：メインメニュー
- Select：持ち物
- A + 十字キー：方向指定モード
- B + 十字キー：高速移動または向き変更

## 斜め移動

Rogueでは8方向移動が重要なので、削除しないでください。Game Boyでは次の候補を検討し、実装しやすく誤操作が少ないものを選んでください。

案A：
- Aを押しながら十字キーで方向指定
- 斜めは方向指定メニューで選ぶ

案B：
- StartまたはSelectで8方向パレットを出す
- 十字キーで方向を選びAで確定

案C：
- 斜め入力は十字キー同時押しに対応する
- 実機で安定しない場合は補助入力を用意する

## 高速移動

通路移動が苦痛にならないように高速移動を用意してください。ただし、敵、アイテム、分岐、罠、部屋入口では止まること。

## 足踏み

その場で1ターン待つ操作を用意してください。候補はB+A、またはメニューから「待つ」。

## メニュー

メニューは軽く、深くしすぎないでください。

- 持ち物
- 装備
- 使う
- 読む
- 食べる
- 投げる
- 捨てる
- ログ
- 中断

## 誤操作防止

死亡や貴重品使用につながる操作は、必要なら確認を入れる。ただし確認が多すぎてテンポを壊さないこと。

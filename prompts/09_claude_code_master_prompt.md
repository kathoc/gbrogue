# Prompt 09 — Claude Code Master Prompt

以下をClaude Codeに最初に貼り付けてください。

---

Mac上でGBDK-2020を使い、実機Game Boyで動くRogueクローンを作るプロジェクトを開始してください。Palm PilotのiRogueに近い体験を目標にし、参照仕様としてUNIX Rogue 5.4系を使います。ただし完全移植ではなく、Game Boyの制約に合わせた再実装です。

最初はASCII/記号タイル表示で作り、後から8x8ドット絵に差し替えられるよう、ゲーム内部IDと表示タイルIDを必ず分離してください。ローグライク風に簡略化せず、未識別アイテム、空腹、呪い、罠、永久死、ターン制、部屋と通路、階層移動、限られた情報で判断する構造を維持してください。

画面はGame Boyの160x144、8x8タイル、20x18マス前提です。下1行をステータス、その上1行を直近メッセージ、必要時に2〜4行のメッセージポップアップ、残りをスクロールするダンジョン表示にしてください。全部屋を1画面に表示する必要はありません。

操作感は風来のシレンGBに近づけてください。十字キー移動、A決定/攻撃、Bキャンセル、Startメニュー、Select持ち物を基本にし、斜め移動は必ず何らかの形で実装してください。高速移動や足踏みも検討してください。

開発環境はMacです。まずGBDK-2020のセットアップ、Makefile、最小ROM、リポジトリ構成、docsを作ってください。次にアーキテクチャを文書化し、段階的に実装してください。各段階でROMがビルドできる状態を維持してください。

mallocは禁止。固定長配列を基本にし、Game BoyのWRAM制約を意識してください。再帰や大きなスタック使用も避けてください。コンパイル警告を放置しないでください。

まず以下を作ってください。

- docs/setup-mac.md
- docs/architecture.md
- docs/status.md
- Makefile
- src/main.c
- src/ 以下の基本モジュール
- assets/ 以下のASCIIタイル管理
- build/ にROMが出る構成

最初のゴールは、Game Boyエミュレータで起動し、タイトル画面から固定マップに入り、ASCIIタイルのプレイヤーを動かせるROMを作ることです。

実装前に、作業計画を短く示してください。その後、ファイルを作成し、ビルド方法を説明してください。

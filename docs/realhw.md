# 実機確認チェックリスト(M11)

ヘッドレス環境では実機確認ができないため、以下を実機で手動確認する。

## 準備

1. `make` → `build/gbrogue.gb`
2. フラッシュカート(EverDrive-GB / EZ-Flash Jr. 等)にコピー
3. カート設定でカートタイプが自動認識されない場合: MBC1 + RAM + BATTERY、
   SRAM 8KB を指定(ROM ヘッダには設定済み: 0x147=0x03, 0x149=0x02)

## チェック項目(GBC)

- [ ] GBC/GBA 実機で黒基調テーマになる(壁=琥珀、アイテム=アクア、
      階段=緑、自機=白緑、敵=橙赤)
- [ ] DMG 実機では従来どおり白基調モノクロのまま
- [ ] GBC 液晶(無バックライト)での暗部の視認性 — 暗すぎる場合は
      render.c の BG_PALS 背景 RGB(1,1,3) を持ち上げる

## チェック項目

- [ ] 起動 → タイトル → START でゲーム開始までが数秒以内
- [ ] 十字移動の入力遅延が気にならない(1歩=1フレーム処理+VBL flush)
- [ ] 初代DMG液晶で `@` `#` `.` `!` `?` などの判別ができる
      (太め5x7フォント。ダメなら scripts/gen_font.py で調整して `make font`)
- [ ] 高速移動(B+方向)中の LCD 残像が許容範囲
- [ ] GFXモード(メニュー→Display mode)の壁/床コントラスト
- [ ] メニュー → Save & quit → 電源OFF → 電源ON → SELECT で再開できる
      (**電池セーブの実機確認はエミュレータでは代替不可**)
- [ ] 再開後にもう一度電源OFF/ON → CONTINUE が消えている(ロード時消費)
- [ ] 死亡 → タイトルに CONTINUE が出ない(永久死)
- [ ] 30分ほど連続プレイしてフリーズ・表示化けがない
- [ ] 電池切れカートでの起動(セーブ無効でも新規ゲームは動くこと)

## 問題が出た場合の当たり所

- 表示化け: render.c の flush は VBL 同期(wait_vbl_done → present)。
  実機で化ける場合は render_present() の呼び出し位置を確認。
- セーブ不良: save.c はチェックサム検証つき。壊れたセーブは自動で
  「なし」扱いになる(起動不能にはならない)。
- 入力: input.c はエッジ+リピート。実機パッドでリピートが速すぎるなら
  REPEAT_DELAY / REPEAT_RATE を調整。

# Architecture

Game Boy (DMG) 実機で動く Rogue クローン。GBDK-2020 / C / 8x8 タイル。
体験の基準は iRogue、ルールの参照は UNIX Rogue 5.4 系。

## 大原則

1. **内部 ID と表示の分離** — ゲームロジックは `tile_id_t` などの内部 ID
   だけを扱う。ID→グリフ変換は `assets/tiles_ascii.c`(のちに graphic
   tileset を追加)だけが行う。
2. **malloc 禁止・再帰禁止** — すべて固定長配列。WRAM 8KB を常に意識。
3. **VRAM 直接アクセス禁止** — 描画は render.c のシャドウバッファ経由のみ。
4. **各マイルストーンでビルド可能** — `make verify` が常に通る状態を保つ。

## 画面 (20x18 タイル)

```
row 0..15   ダンジョンビューポート(スクロール)
row 16      直近メッセージ 1 行(重要イベントは 2〜4 行ポップアップ)
row 17      ステータス: L{階} HP{c}/{m} St{力} AC{n} 状態
```

## モジュール

| ファイル | 責務 |
|---|---|
| `main.c` | 起動、init 呼び出し |
| `game.c/h` | 状態機械(タイトル/プレイ/死亡)、ターン進行。**薄く保つ** |
| `map.c/h` | 論理マップ(タイル+探索/視界フラグ)、アクセサ |
| `mapgen.c/h` | Rogue 風 3x3 セルの部屋+通路生成、階段・罠・アイテム配置 |
| `fov.c/h` | Rogue 式視界: 部屋単位の明かり + 通路は隣接 1 マス |
| `render.c/h` | シャドウバッファ→VBL で行単位 VRAM flush |
| `input.c/h` | エッジ検出、D-pad リピート、コンボ(A+方向 等) |
| `actor.c/h` | プレイヤー+モンスター(固定長配列) |
| `combat.c/h` | 命中/ダメージ/AC/経験値 |
| `items.c/h` | アイテム定義、識別状態、効果 |
| `inventory.c/h` | 持ち物(固定スロット) |
| `rng.c/h` | LCG。状態の get/set(セーブ用) |
| `save.c/h` | SRAM 中断セーブ、死亡時破棄 |
| `ui_*.c/h` | 持ち物画面、メニュー、ログ。**game.c と別翻訳単位に置く** |
| `assets/tiles_ascii*.c` | ASCII アトラス(生成物)+ ID→グリフ表 |

## 描画パス(2モード)

**ワールドモード(通常プレイ)** — スムーズスクロール/スムーズ移動:
- 論理マップは 32x28 で、レベル全体が BG タイルマップ(32x32)に常駐。
  スクロールは SCX/SCY レジスタのみ(タイルストリーミング不要)。
- 下2行(メッセージ+ステータス)は Window レイヤ(0x9C00, WY=128)に固定。
- プレイヤーとモンスターはハードウェアスプライト(OAM 0..12)。
  1歩ごとに旧位置→新位置を 4フレーム×2px で補間(game.c の
  animate_turn)。カメラも同時にピクセル補間。2タイル以上のジャンプ
  (階段・テレポート)はスナップ。
- BG が変わるのは探索での開示・アイテム増減・罠発覚時のみ
  (worldview.c が該当セルだけ world シャドウへ再ペイント)。
- BG/OBJ は 0x8000 非符号アドレッシングでタイル表を共有
  (LCDCF_BG8000)。OBP0/BGP は 0xE4 に明示初期化。

**オーバーレイモード(モーダルUI)** — タイトル/パック/メニュー/ログ/
ポップアップ/死亡・勝利画面は従来どおり 20x18 シャドウバッファ+
スクロール0+Window非表示+スプライト退避で全面描画する。
`render_set_world(0/1)` で切替、復帰時は `view_world_enter()` が全再
ペイントする。

いずれも書き込みは WRAM シャドウ+dirty 行のみで、`render_present()`
が dirty 行を `set_bkg_tiles`/`set_win_tiles` で VRAM へ流す。

**入力**: `input_tick()` がエッジを毎フレームラッチする(アニメーション
や全面再描画中も呼ばれる)ので、短いタップが失われない。

VRAM タイル 0..94 = ASCII 0x20..0x7E。**タイル番号 = 文字コード - 0x20** を
不変条件とする(テストが BG/OAM を文字列にデコードするため)。
VRAM タイル 95.. = グラフィックアトラス(M10)。両方常駐なので表示切替は
ルックアップの切替のみ。スプライトも同じタイル表を参照する。

ROM は 32KB フラット(MBC1 だが bank1 を常時マップ=線形アドレス)。
32KB を超える場合はデータ表からバンク退避する(status.md 参照)。

## テスト戦略

`tests/gbtest.py` が PyBoy をヘッドレスで起動し、

- BG タイルマップ → 20x18 の ASCII 文字列にデコードして assert
- `build/gbrogue.noi`(no$gmb 形式)から WRAM シンボルを引いて直接 assert
  (マップ全体を読み Python 側で BFS 到達性まで検証する)
- ボタン列をスクリプト実行、PNG スクショも保存
- `make debugrom`(-DGBR_DEBUG_KIT)は既知の試験アイテム一式と短い空腹
  ゲージで起動し、M7/M9 系を決定的にテストする

各マイルストーンに `tests/verify_mN.py`。`make verify` で全実行。

## 既知の SDCC/sm83 制約(前プロジェクトからの教訓)

- `game.c` 内に描画呼び出しサイトを増やしすぎるとブートパスが壊れる
  リンカ配置問題が観測されている。UI 描画は必ず別翻訳単位に分ける。
- `DISPLAY_OFF` 中の `setchar` 系はハング。使わない(render.c 経由のみ)。
- 起動直後は `input_swallow_edges()` で幻のエッジを飲み込む。

## メモリ予算(WRAM 8KB)

| 用途 | サイズ |
|---|---|
| 論理マップ 32x28 ×1B(地形5bit+探索/罠隠蔽フラグ) | 896B |
| ワールドBGシャドウ 32x28 + Window 2x20 | 936B |
| オーバーレイシャドウ 20x18 | 360B |
| アクター(プレイヤー+モンスター16体) | ~300B |
| 床アイテム 24 個 | ~150B |
| 持ち物 16 スロット | ~80B |
| 識別テーブル(薬/巻物/杖/指輪) | ~80B |
| メッセージログ 16×20 | 320B |
| 予備(スタックは HRAM 側 + 上位 WRAM) | 残り |
